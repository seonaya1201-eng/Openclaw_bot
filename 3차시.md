# OpenClaw School 3차시 — 개인비서에서 직원으로 전환하기

## 차시 개요

- **주제:** 단순 보조를 넘어 스스로 판단하고 실행하는 'AI 직원' 만들기
- **핵심 키워드:** SOP(Standard Operating Procedure), Tool Use, Proactive Action, Autonomy
- **목표:** "지시를 기다리는 비서"에서 "워크플로우를 완수하는 직원"으로 에이전트의 역량을 격상시킨다.

---

## 강의 목표

단순한 질문 답변 수준을 넘어, 정의된 업무 절차(SOP)에 따라 도구를 활용해 독립적인 결과물을 만들어내는 **AI 실무자**를 기획하고 구현한다.

## 학습 목표

- 비서(Assistant)와 직원(Employee)의 기능적 차이를 이해한다.
- 업무 매뉴얼(SOP)을 프롬프트 아키텍처로 변환하는 법을 배운다.
- 브라우저 제어와 API 연동을 통해 에이전트의 업무 범위를 현실로 확장한다.
- 에이전트가 처리한 결과에 대한 보고 및 검토 프로세스를 설계한다.

---

## 주요 내용

### 1. 비서에서 직원으로: 마인드셋의 변화

- **수동적 응답 vs 능동적 해결**: 질문에 답만 하는 것이 아니라, 목표를 달성하기 위한 다음 단계를 스스로 제안
- **업무 범위(Scope)의 설정**: 이 에이전트가 책임지고 완수해야 할 업무의 시작과 끝을 정의
- **권한 부여**: 에이전트가 브라우저를 켜거나 메일을 보낼 수 있도록 실질적인 도구 권한 부여

### 2. 업무 매뉴얼(SOP)의 프롬프트화

좋은 직원은 매뉴얼을 따릅니다. 프롬프트를 단순한 지시문이 아닌 **'업무 표준 절차서'**로 작성합니다.

| 단계 | 프롬프트 구성 요소 | 설명 |
|-----------|--------------------|------|
| **상황 판단** | Trigger Recognition | 어떤 상황에서 이 업무를 시작해야 하는지 정의 |
| **단계별 절차** | Step-by-Step SOP | 1단계 지식 검색, 2단계 외부 도구 확인 등 순서 정의 |
| **예외 처리** | Error Handling | 정보가 부족하거나 도구가 작동하지 않을 때의 대처법 |
| **최종 검수** | Quality Control | 발송 전 스스로 체크해야 할 리스트 제공 |

### 3. 실전 도구 활용 (Employee Skills)

에이전트에게 필요한 실무 도구를 장착합니다.

- **브라우저 릴레이(Browser Relay)**: 웹사이트에서 실시간 정보 수집, 예약 결제, 게시물 등록 등 수행
- **Maton API (Professional)**: 캘린더 일정 관리, 지메일 기반 고객 대응, 시트 데이터 기록 자동화
- **멀티 에이전트 협업**: 리서치 담당 직원과 보고서 작성 담당 직원의 연동(Concept)

### 4. 보고 및 성과 관리 (Feedback Loop)

직원은 결과물을 보고해야 합니다.

- **업무 결과 보고**: 수행한 작업의 로그(Log)를 요약하여 메신저로 전송
- **피드백 및 고도화**: 사람이 수정한 내용을 다시 프롬프트나 지식베이스(KB)에 반영하여 학습
- **지속적 개선**: 에이전트의 업무 처리 성공률을 모니터링하고 SOP를 업데이트

---

## 실습

1. **업무 정의서 작성**: 에이전트가 담당할 '직무 명세서' 작성 (예: "주간 업계 동향 리포터")
2. **SOP 기반 프롬프트 구축**: 2차시의 프롬프트를 업무 절차서 기반으로 리모델링
3. **도구 연동 및 실행**: 브라우저나 API를 하나 이상 연결하여 실제 작업을 완수하는지 확인
4. **결과 보고 자동화**: 작업 완료 후 요약 리포트를 나에게 전송하도록 설정

## 과제

1. **업무 자동화 시나리오 확장**: 현재 에이전트가 하는 일의 전후 단계를 추가하여 '프로세스' 만들기
2. **예외 처리 시나리오 3개 추가**: "비밀번호가 틀렸을 때", "정보가 없을 때" 등의 대처 프롬프트 보강
3. **직원 평가**: 에이전트가 만든 결과물 5개를 평가하고, 개선 포인트를 SOP에 반영하기
